Expense Management

Submit a receipt, route it to the right approver, watch it move from pending to reimbursed — and run the whole thing on cloud services that scale up under load and cost almost nothing when idle. I rebuilt this expense system from scratch because it's the cleanest way to learn cloud architecture: a real app, designed around managed services instead of a server you babysit.

What it is

An app designed for the cloud, not just hosted on it

The feature set is familiar — employees file expenses with receipts, managers approve or reject, finance reimburses. What makes it a cloud project is the architecture. Instead of one server running everything, the work is spread across managed building blocks: a serverless function for the logic, a managed database for the records, object storage for the receipt images, and a hosted auth service for logins.

That shift is the whole lesson. You stop thinking about machines and start thinking about services that someone else keeps running, scaling and patching. The skill becomes wiring them together cleanly — and understanding the trade in control for elasticity and operational calm.

The stack

From receipt to reimbursement, on managed services

Each concern handed to the service built for it.

Object storage

Keep receipt images in scalable blob storage, separate from the records that reference them.

access

Hosted auth

Offload logins, sessions and roles to a managed identity service.

workflow

Approval routing

Move each expense through submitted → approved → reimbursed with the right approver.

repeatable

Infrastructure as code

Define the whole stack in code so it can be torn down and recreated identically.

Architecture

How an expense flows through the cloud

A single claim touches several managed services in turn:

  1. Submit

    An employee logs in via the auth service and files an expense with a receipt.

  2. Store

    The receipt goes to object storage; the record goes to the managed database.

  3. Route

    A serverless function assigns it to the correct approver.

  4. Decide

    The approver accepts or rejects, updating the expense's state.

  5. Reimburse

    Approved expenses move to reimbursed and close the loop.

Reflection

What rebuilding it taught me

  • Cloud is a design choice, not a host. The win comes from architecting around managed services, not from renting a VM.
  • Managed services trade control for calm. You give up tuning the database yourself and get backups, scaling and patching in return.
  • Storage and data want separating. Receipts belong in object storage; their metadata belongs in the database — mixing them is a classic mistake.
  • Scale-to-zero changes the economics. Serverless makes a bursty, occasionally-used app genuinely cheap to run.
  • Infrastructure as code is non-negotiable. If you can't recreate the stack from a file, you don't really understand it.
